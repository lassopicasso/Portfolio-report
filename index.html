Semester Project 1

Deployed before changes: 28 February 2021

Website Github

What Was Changed

  1. Changed text-shadow on homepage images texts
  2. Moved li's outside of anchors
  3. HTML corrections on social media icons
  4. Made the CSS more DRY
  5. Corrected URLs
  6. Add validation to contact form

How I changed It

(1) CSS: Instead of having 1 text shadow, I added 4 text shadows; above, below, left and right. Behaves like a text border.

(2) HTML: Moved anchors from being outside to inside of li in dropdown menu. CSS: To make the container(li) clickable and not only the text, displayed anchor as block, where width and height is 100%

(3) HTML: Removed label wrapper for social media anchors in footer.

(4) CSS: Many HTML elements got same/similar styling but with different classes. Changed this so they have same classes. For example: elements in input form and donate form, elements with same color, images with same styling.

(5) HTML: Removed forward slashes in front of the URLs

(6) HTML: Added error message below each input field. CSS: Error message is displayed as none as default. Added a new class cta-opacity that gives the "send button" a style of 50% opacity and no-hover effects. It occurs as disabled until each input is correct. JS: When user unfocus an input field, the JS script checks if the input is validated or not. It also continuously control if every input field is validated. If the unfocused input field is not correct an error message displays. If all input fields are correct, the "send button" is no longer disabled.

Why I make these changes

(1) Added "text border" on the text on images because of feedback from tutor that it was not so readable.

(2) The only legal element allowed inside an "ul" is li. Anchor cannot be wrapped around li element. Therefore, it has to be inside the li element.

(3) Unnecessary to wrap label around social media anchors. They don't contribute to anything.

(4) Making the CSS more DRY. Easier to do changes. Reduced the CSS script with approx 60 lines, but still much potential for a DRYer CSS.

(5) Unnecessary to have forward slashes in front the URLs. Feedback from tutor.

(6) Makes the website more interactive and the user probably experience a better user journey where he/she can be sure that the inputs were correct when sending.

Cross-Course Project

Deployed before changes: 22 August 2021

Website Github

What Was Changed

  1. Aligned and resized items in navbar
  2. Reduce distance between product images and their grid header.
  3. Added content to footer
  4. Adjusted distance for elements and containers in the pages product details and checkout
  5. Made the search input in NAV-bar functional
  6. Make javascript codes run more efficient
  7. Removed redundant images stored locally

How I changed It

(1) CSS: Resized the search icon from 24 to 26px, gave the placeholder 1em and aligned-items center for the items in the container.

(2) CSS: Removed the margin 20px that I had added on the images inside the grid. Not sure why It was added a margin there in first place. Unnecessary to have a margin-top on the images in grid when the container have a margin-top as well.

(3) HTML: Replaced the information icon (which was the only element in footer) and added divs containing the elements company info, links and social media icons. CSS: Displays the divs as flex with a space-around justify-content and a max-width equal to main container.

(4) CSS: For the checkout page I removed the top and bottom margin for H1, also added margin for main container to the navbar and footer. The specific product page add display height 100% to the main.

(5) HTML: Added an empty div in the search form, where any given match on input displays in a dropdown list. CSS: The dropdown list of search results have a position absolute so I can place the dropdown list below the search input box. JS: Created 2 arrays, one contains only the product titles and one contains all data in API. When an "keyup" interaction occurs it calls the function showResults. This function first controls input length, if the length is more than 0 it calls a second function that filters the title array. Any titles that matches is returned to the first function and collected in a new array. The titles from this array are being displayed in a dropdown list wrapped in an anchor which redirects the user to product details page if clicked on. However, if no titles is found, following message is displayed: "No products found".

(6) JS: Instead of having two JS-files, one for women.html and one for men.html, I merged them together with minimal extra lines (3 lines). In function getProducts I add an if statement on window.location.href to check what html page is visited and stores the response in a variable that is used to gather data. Merged product-details.js and imagesSlider.js, both used in product-specific.html. Index.js created a separate function for innerhtml, so that there are no need to repeat the same script twice.

(7) Went through local image folder and removed those that could rather be stored in API.

Why I make these changes

(1) Adjusted the items in the navbar because they didn't seem to be inline and with different sizes.

(2) It didn't look good. It was too much distance between the images and their header above them.

(3) The footer looked empty and not finished. Therefore I added more elements to the footer.

(4) For checkout page there was too much space above and below the H1. When the viewport got narrower, there wasn't any margin for the container to the navbar and footer. To make this page more consistent with the other pages I added margins. For the product specific page, when the viewport was large enough the background color in main was only surrounding the container. So there was a gap between navbar/footer and the background-color in main.

(5) If there are many products in the online store, and a potential customer is looking for a specific product, the search function facilitates the search of this product.

(7) Reduce the size of the website. Makes it go faster.
